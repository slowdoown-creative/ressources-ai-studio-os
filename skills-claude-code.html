01 — Concept

C'est quoi
un skill ?

Claude a une mémoire limitée. Si tu lui donnes trop d'instructions en même temps, il ralentit et fait des erreurs. Un skill résout ce problème en ne chargeant que ce dont il a besoin, au moment où il en a besoin.

🧠

Claude sans skill

Il doit tout mémoriser en permanence. Sa mémoire sature. Il devient lent, fait des erreurs, peut crasher sur les longues sessions.

VS
📚

Claude avec skills

Il dispose d'un classeur. Il ouvre uniquement la fiche recette dont il a besoin, au bon moment. Le reste reste rangé.

💡

La règle d'or : Un skill n'est jamais chargé en permanence. Claude lit d'abord la description de chaque skill, puis décide seul lequel utiliser selon ce que tu lui demandes. C'est pourquoi bien écrire la description est la chose la plus importante.

02 — Taxonomie

Les 3 types
de skills

Chaque skill a un rôle précis. Comprendre ces trois catégories te permet de savoir exactement quel skill créer selon ton besoin.

TYPE 01
🔧
Outil
Maîtriser un logiciel externe

Tu donnes à Claude le mode d'emploi d'un outil. Il sait alors comment l'utiliser de façon autonome, sans que tu aies besoin d'expliquer à chaque fois.

→ Contrôler un navigateur web
→ Créer une page dans Notion
→ Rechercher dans une base de données
TYPE 02
⭐
Compétence
Améliorer la qualité

Tu donnes à Claude tes bonnes pratiques et ton style personnel. Il produit un résultat de meilleure qualité, qui te ressemble vraiment.

→ Écrire dans ton style LinkedIn
→ Générer une description YouTube
→ Rédiger un post Circle
TYPE 03
⚙️
Workflow
Suivre un processus

Tu définis un processus en plusieurs étapes que Claude doit suivre dans l'ordre strict. Il ne peut pas sauter d'étape ni improviser.

→ Créer un script vidéo YouTube
→ Débugger un code en 5 étapes
→ Analyser et reformater du contenu
🔗

Les skills peuvent s'empiler

Un skill workflow peut appeler d'autres skills en cours de route. Exemple : un skill "Script YouTube" qui utilise automatiquement le skill "Notion" pour écrire le résultat, puis le skill "Excalidraw" pour créer le schéma. Comme des Lego.

03 — Structure

Comment est
construit un skill ?

Un skill, c'est simplement un dossier avec des fichiers texte. Rien de compliqué. Le fichier principal est la table des matières — les autres ne s'ouvrent que si nécessaire.

Structure d'un skill
📂 mon-skill/
📄 skill.md Obligatoire ← table des matières
📝 instructions.md Optionnel ← détails
🗂 template.md Optionnel ← modèle
⚡ script.js Optionnel ← code

skill.md — Toujours chargé

C'est le cerveau du skill. Il contient la description qui dit à Claude quand l'utiliser, et les chemins vers les fichiers annexes. Volontairement court (~400 tokens).

Fichiers annexes — Chargés si besoin

Instructions détaillées, templates, code. Claude décide seul s'il en a besoin selon la tâche. Si non, ils ne sont pas lus. La mémoire reste légère.

📌

Un skill bien optimisé comme dans la vidéo fait 137 lignes dans son fichier principal — soit ~400 à 600 tokens. Presque rien.

1

Tu parles à Claude

Tu tapes une commande ou tu parles naturellement. En arrière-plan, Claude a déjà lu les descriptions de tous tes skills.

/script-youtube "Les skills dans Claude Code"
2

Claude identifie le bon skill

Il compare ta demande avec les descriptions disponibles et choisit lequel activer. Il charge le skill.md — la table des matières uniquement.

3

Il ouvre uniquement ce dont il a besoin

Si la tâche nécessite le template Notion, il l'ouvre. Sinon, ce fichier n'est pas chargé. La mémoire reste légère et l'agent reste performant.

✓

Il exécute avec précision

Avec exactement les bonnes instructions, ni plus ni moins. Résultat ciblé, mémoire optimisée, session propre.

04 — Pratique

Les commandes
essentielles

Deux façons de déclencher un skill — par commande directe ou en langage naturel. Et une commande pour tout remettre à zéro.

/nom-du-skill
→

Déclenche un skill directement par son nom. Le moyen le plus rapide et le plus précis.

"crée mon script YouTube"
→

Claude détecte le mot-clé dans ta phrase et charge le bon skill automatiquement, sans commande explicite.

/new  ·  /clear
→

Vide la mémoire de Claude et décharge tous les skills actifs. Idéal avant de démarrer une nouvelle tâche.

/help
→

Affiche tous les skills disponibles et leurs descriptions pour savoir ce que Claude sait faire.

Résumé

3 choses à retenir

01
La description est tout

C'est elle qui dit à Claude quand utiliser un skill. Si elle est floue, Claude ne sait pas s'en servir. C'est l'élément le plus critique à soigner.

02
Chargement à la demande

Un skill ne charge que ce dont il a besoin. Fichier principal toujours, fichiers annexes seulement si la tâche l'exige. La mémoire reste légère.

03
Les skills s'empilent

Un skill peut appeler d'autres skills. Tu assembles des workflows complexes avec des blocs simples — comme des Lego qui se connectent entre eux.
